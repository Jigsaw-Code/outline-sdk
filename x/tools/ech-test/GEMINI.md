# Project: ECH Analysis Tools

## Project Overview

This project provides a suite of tools for analyzing the deployment and impact of DNS HTTPS resource records (RRs) and Encrypted ClientHello (ECH). The primary goal is to gather data on DNS latency, service support for ECH and related standards, and potential network interference.

The project is composed of two main Go-based command-line tools:

1.  **`dnstest`**: Performs large-scale DNS analysis by querying a list of top domains (from the Tranco list) for A, AAAA, and HTTPS records. See `dnstest/README.md` for more details.
2.  **`greasetest`**: Tests ECH GREASE compatibility by issuing HEAD requests to top domains with and without ECH GREASE enabled, using a custom ECH-enabled `curl` binary. It also generates a report summarizing the findings. See `greasetest/README.md` for more details.

Additionally, the project includes Python scripts for processing and visualizing the data generated by these tools.

## Building and Running

The tools are located in their respective subdirectories (`dnstest/` and `greasetest/`) and are designed to be run as Go programs from the `ech-test` directory.

### `dnstest`

To run the DNS analysis tool:

```sh
go run ./dnstest --topN 100
```

This command will query the top 100 domains from the Tranco list and save the results in `./dnstest/workspace/`.

### `greasetest`

To run the ECH GREASE compatibility tester:

```sh
go run ./greasetest --topN 100
```

This command will perform HEAD requests to the top 100 domains and save the latency and error results in `./greasetest/workspace/`.

### Plotting Scripts

The Python scripts for plotting the results are located in the `dnstest/report/` directory. They require `pandas`, `seaborn`, and `matplotlib`.

To run the plotting scripts (from the `dnstest/` directory):

```sh
python report/plot_durations.py <input_csv_file> <output_png_file>
```

## Development Conventions

*   The project follows standard Go coding and formatting conventions.
*   The tools are self-contained and can be configured via command-line flags.
*   The `dnstest` tool uses the `github.com/miekg/dns` library for DNS queries.
*   The `greasetest` tool uses the standard `net/http` and `net/http/httptrace` packages for making HTTP requests and gathering latency metrics. It currently contains a placeholder for enabling ECH GREASE, which will require a custom `http.Transport` with a `tls.Config` that supports ECH. The `greasetest/README.md` contains instructions on how to build a custom `curl` with ECH support.
*   Both tools rely on the Tranco list for the domains to be tested.
*   Output data is consistently generated in CSV format for easy analysis.

### Python Environment

When running Python scripts or installing Python packages, it is recommended to use a virtual environment to avoid conflicts with global Python installations.

To set up a virtual environment:

```sh
python3 -m venv <path/to/venv>
source <path/to/venv>/bin/activate
pip install <package_name>
```

Remember to activate the virtual environment before running any Python scripts or installing packages.
